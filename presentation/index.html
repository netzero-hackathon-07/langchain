The Problem

AI를 쓸수록
탄소가 쌓인다

  • 사용자는 단순 작업도 습관적으로 가장 비싼 고성능 모델로 처리한다
  • 이메일 다듬기, 번역, 단순 질문까지 전부 GPT-4o · Claude Sonnet으로
  • 단순 작업에 고성능 모델 = 불필요한 토큰 = 불필요한 전력과 CO₂
  • 생성형 AI 사용량이 폭증하는 지금, 낭비는 매 요청마다 누적된다

Our Solution

일마다, 맞는 모델에게

  • 복합 요청을 여러 서브태스크로 분해
  • 각 작업의 성격(난이도·유형)을 판단
  • 단순 작업은 저비용·저탄소 모델, 복잡한 작업만 고성능 모델
  • 전체를 단일 고성능 모델로 처리한 경우 대비 비용·탄소 절감

Under The Hood

저비용 모델이
직접 라우팅한다

  • Claude Haiku 4.5가 요청을 분석해 서브태스크로 분해하고 모델을 추천
  • 라우팅 판단마저 저비용 모델이 담당 — 효율 컨셉에 완벽히 부합
  • 각 스텝을 실제로 실행 (mock 아닌 진짜 AI 응답)
  • 실측 토큰 기반으로 비용·CO₂ 계산 — 추정이 아닌 실제 수치
